- 在充分阅读后端 Profile/Pipeline/Playback/Worker/SessionImport 全链路代码和前端对应视图的基础上，设计了"时间对齐→帧序列编码→质检"三步骤 Pipeline 链

### 2. 后端新增 AssetType 与规则实现

- `AssetType.java` 新增 `IMU_ALIGNED_CSV`、`ALIGNMENT_REPORT`、`QC_SUMMARY` 三个枚举值
- 新建 `FakeStereoParserRuleResolver.java`：`@Component`，supports `FAKE_STEREO_SESSION_V2`，子串匹配 sourceKey 和 filePattern
- 新建 `FakeStereoPlaybackRuleResolver.java`：`@Component`，supports `FAKE_STEREO_PLAYBACK_V2`，`buildSources()` 构建视频+IMU 双源，`canPlay()` 要求两者同时满足
- 通过 Spring `List<>` 注入自动注册到 `ProfileRuleRegistry`，零配置

### 3. 3 个 Worker Pipeline 处理脚本

- `stereo_imu_align.py`（STEREO_IMU_ALIGN）：读取 `frame_timestamps.csv` 获取帧时间戳，读取 `imu_data.csv` 获取 IMU 序列，计算三路共同时间窗口，裁剪左右帧到 `aligned/` 子目录，IMU 数值列线性插值到每帧时间戳，输出 `alignment_report.json`
- `build_stereo_mp4.py`（BUILD_STEREO_MP4）：按 sourceKey 分组，优先使用 `aligned/` 子目录中的对齐帧，ffmpeg `%06d.jpg` 模式合成 MP4，失败回退 concat 模式
- `build_playback_bundle.py`（BUILD_PLAYBACK_BUNDLE）：ffprobe 获取视频元数据，执行四项 QC 检查（帧数一致性、IMU 时间覆盖、行数匹配、FPS 偏差），综合评分 → PASSED/WARNING/FAILED
- 均继承 `BasePipeline`，经 Worker `--list-pipelines` 验证 4 个 Pipeline（含旧 BUILD_PLAYBACK）全部自动注册成功

### 4. 造数脚本改造

- `make_fake_stereo_imu_session.py` 的 manifest 从旧版数组格式升级为 backend `SessionImportServiceImpl.normalizeManifest()` 接受的 modern 格式（`schemaVersion: "2.0"`、`task.profileCode`、`sources` 为 key→{type, path} 对象）
- 输出目录名从 `stereo_left_frames/stereo_right_frames` 统一为 `left_frames/right_frames`，与 Profile Source 的 sourceKey 对齐
- `frame_timestamps.csv` 路径统一使用 Unix 正斜杠（`Path.as_posix()`）
- Windows GBK 终端编码兼容处理（emoji→ASCII 标记 + stdout UTF-8 reconfigure）
