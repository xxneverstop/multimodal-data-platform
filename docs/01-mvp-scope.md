# 第一阶段 MVP 功能范围

## 目标

第一阶段目标是搭建一个可运行的项目骨架，完成“采集任务创建 → 文件上传 → OSS 存储 → 元数据入库 → 基础质检 → 报告展示”的闭环。系统不追求一次性支持所有模态，先以 IMU 数据为例，保证后续可以继续扩展。

## 必须实现的功能

1. 采集任务管理：支持创建、查询采集任务。任务中需要记录任务名称、被试编号、动作名称、设备类型、采集日期、备注等信息。
2. 文件上传：支持上传 IMU 文件，文件可以是 csv、txt、json、bvh、fbx 等格式，第一阶段不强制解析复杂格式，只做基础格式识别和元数据记录。
3. OSS 存储：上传后的原始文件保存到 OSS，数据库只保存文件路径、文件名、大小、类型、关联任务等元数据。
4. 基础质检：上传后可以触发基础质检，检查文件是否为空、文件大小是否合理、扩展名是否支持、是否能读取基本行数或 JSON 结构、是否存在明显缺失。
5. 报告展示：质检结果以 JSON 形式存入数据库，并提供接口查询。报告内容包括总体状态、通过/失败项、警告信息、文件基本信息。

## 暂不实现的功能

第一阶段暂不做复杂权限系统、多人协作审批、完整前端大屏、复杂 IMU 姿态解算、多模态时间同步、数据集版本管理、异步工作流引擎、算法训练平台、三维可视化。可以预留字段和接口，但不要过度设计。

## 技术栈建议

后端使用 Spring Boot 3、MyBatis-Plus、MySQL、OSS SDK。前端可以先不做复杂页面，接口优先，可以用 Apifox 测试。后续再补 Vue 管理界面。基础质检第一阶段可以直接写在 Java 服务中，后续再扩展为 Python worker 或异步任务。
