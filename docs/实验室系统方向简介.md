# 它长什么样子？（What Looks Likes）

简单说，就是：衣服+眼镜+手套

![](https://cdn.nlark.com/yuque/0/2026/png/49050880/1778637402250-6a3d2caa-f4ce-4bd5-b0c3-e562bd7f91d2.png)

衣服，显然是我们的衣服

眼镜，预期是双目RGB+IMU，用于采集第一视角的画面，并从中重建出手物交互、环境信息；

手套：预期是11个IMU，同时结合第一视角的画面，能够进一步稳定、高精度重建出手物交互；这个预期半年时间内拿出系统，接下来就是做触觉sensing。

衣服，我们经常被别人评价是：interesting，但价值不知道如何体现。我们也去提机器人数据采集，但虽然大家都模糊地认为这样的数据是有价值的，但大家有1个共识+1个顾虑。共识是只有人的大关节肢体运动数据是不够，从机器人而言，大家更关心两个信息：一是手部的操作，二是第一视角画面。

其实严格上来说，这个系统搭建也没那么复杂，有做衣服的经验，我对搭建这个系统充满信心。

没有完全抛开衣服，我们也不可能完全抛开衣服。

可以更为灵活的形式，为了不同的项目交付，我们可以替换为绑带式的，也可以把衣服替换为视觉方案去捕捉人的肢体动作。这都没有问题，但我们要时刻清醒地意识到，这只是为了服务项目交付，而不是我们真正的系统！**我们的系统是让我们显著性地区别于其他研究团队/公司，集合团队所有力量、值得长期打磨的**。

# 我们要拿它解决什么问题？（For What）

我们做这套系统，不希望再重蹈覆辙，造了一个大锤子，但不知道钉子是什么。

我们做这套希望，目标就是采集用户在执行日常任务中的数据，提供给具身智能机器人用于训练。

前面说了1个共识是只有人的肢体动作是不够的，需要有手部精细操作和第一视角的图像。

1个担忧是这样的无本体方式采集出来的数据，是否有价值？对于机器人操作的算法模型是否真的有性能的提升？

如何去回应这个担忧呢？

最简单的方式是我们拿着这个数据去训练模型，直接在最后的任务上去验证任务完成的效率和成功率，就像安踏的那个任务。但毫无疑问，这需要有大量的计算资源和一流的人才。

最省心的方式是把数据提供给别人，我们依然只定位在数据采集的环节，只做一个卖铲子的人。定位明确，但需要有其他人来证明这个铲子是有用的。

这两个方式不矛盾，应该要同时结合起来做。**但我们要慎重审视自己独立开展模型学习的能力**。

# 我们要围绕它做什么事情？（What to do）

算法层面显然是一个重要的事情，如何做的更准确？但这里面也有很多事情是要评估要做不要做的，是不是我们做最合适的。

比如双目相机去做深度重建，这种最好就用现成的框架，先用好再说，这本身是CV领域的老话题，你很难做好，做好也不无法在学术圈留下痕迹，所以就不要做。

比如双目相机去做手物交互，这种最好就交给清华徐枫老师去做，我们两个团队关系好，他之前就做了手物交互，让他们去做；

真正适合我们做的事情是什么：是系统本身的构建。比如系统的准确性，比如利用双目相机和手部IMU去提升手部姿态的重建，利用双目相机和全身去提升全局移动的精度。

大家一定要树立的概念，是系统观念，“利用双目相机和全身去提升全局移动的精度”不是这一句话就只能发一篇A，不要担心好像人多了就这么点事情，没有充足的空间分配给大家。如果老师这样想，说明老师本身思维有限制、不够勤奋去思考主题；如果同学们这样想，那就要改变观念，紧密围绕在团队的系统这棵大树，获取尽可能多的资源，快速形成高质量论文发表的节奏。

要像之前做衣服一样，从里面挖出一系列的A，科研问题要来自系统，系统要支持高水平论文。

为什么我在这里谈的主要是让系统做的更好，而没有去谈验证数据的价值

这个事情，我和安军、姚远、徐枫老师他们在交流的时候，打过一个比喻：主力部队和轻骑兵。大部分人应该仍然团结在核心系统的搭建上，少部分能力突出的同学可以作为轻骑兵去开展工作，挑战更高难度的任务，比如拿数据去验证模型的效果。

但不是所有同学都有这个实力，要跑到具身智能模型这个赛道找饭吃，这都是世界级的人才在竞争的事情，所需要的计算资源、算法研究能力是不可低估的。没有这个实力，我不建议同学们考虑这个方向。你跑不过其他团队，我们团队也没有火箭帮你跑过其他团队，给你提供一匹马是可以的，能不能做出来这靠的是个人的能力（造化）。没有超强的能力，一般是做不出来的，到那个时候就是相互埋怨。

# 你要如何参与其中？

既然主线定了，那么每个人就要放到这里面去。我们依然是执行 系统-科研-项目/硬件-算法-软件的矩阵式团队模式，那大家就要想清楚，我在这里面是什么角色（硬件、算法、软件）？，做的工作产出是什么（系统、科研、项目）？大家在思考自己角色的时候（硬件、算法、软件？），多想想自己能做什么事情、具备什么能力；大家在想自己工作产出的时候（系统、科研、项目），多想想自己以后要做什么，需要具备什么条件。

这里面有一些事情和大家同步的，方便更多的同学来选择自己的工作内容。

衣服一直在做，最近由福州的裕元公司来帮助生产，这是一件好事情，可以极大减轻诗凡的工作量，也能提升我们的产能。

眼镜由钦泽在做，摄像头已经选型了，最差的方案是拿一个开源项目去魔改，这里面需要有3D建模和打印的技能，如果有人有这个技能包，欢迎加入。更高端一点的，我在找供应商来配合我们，有则最好、无则照样推进。

手套的11个IMU版本也在做，两周时间应该是可以拿到硬件。我们还在物色手套的集成商，需要有人来配合和工厂的沟通，真正把手套做出来。

可以预见的是，眼镜和手套都有很多的沟通要做，才能真正达到衣服现在的水平，这个是系统组要做的事情。

我们显然需要有一个数据采集系统，去把这些数据都收集起来，最好是个平台，除了储存数据以外还能有一些其他打标签、分享的功能。这就是开发的工作。

我们显然还有很多算法的工作要做，让手指跟踪更准确、让全身跟踪更准确，先把这两个事情做好，然后再考虑是不是要做手物交互、环境重建、世界模型这些。

把这些事情列出来，这样工作才能具象化，每个人在这里面参与什么，目标是什么，就更为清晰。接下来，就是每个人自己的选择；你自己不选，那么就是被安排。

当然，我要说的是，因为我们团队在过去一年的时间里面处于发散状态，现在要收回来，这必然会导致有的同学手上还有正在进行的活。有的是可以继续做下去的，有的是要做调整的。
