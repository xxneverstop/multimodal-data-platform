# 多模态数据基础设施平台（MMDP）项目概述

## 项目背景

随着机器人与具身智能方向的发展，实验室与行业中的数据采集场景正在快速复杂化。当前的数据类型已经不再局限于单一视频，而逐渐扩展为多模态协同数据，包括 RGB 视频、深度数据、IMU、动作捕捉、SMPL、MCAP、HDF5，以及未来可能接入的 EEG、足压与机器人状态数据。

然而当前实验室与多数机器人数据采集团队仍主要依赖“硬盘 + 文件夹 + 网盘 + 人工记录”的方式管理数据，普遍存在以下问题：

- 数据分散，缺少统一管理；

- 无法追踪数据来源与处理过程；

- 缺少统一时间轴与多模态关联；

- 数据处理链路不可复现；

- 缺少统一质检与状态管理；

- 原始数据与派生结果关系断裂；

- 后续模型训练与数据集构建困难。

尤其在多模态场景下，一次真实采集往往同时产生：

```text
video
imu
csv
smpl
alignment
qc
annotation
```

如果缺少统一的数据基础设施，后续的数据治理、算法开发与数据集建设将难以持续扩展。

因此，本项目旨在构建一套面向机器人与具身智能场景的多模态数据基础设施平台，实现从数据采集、数据接入、数据资产管理、处理流程、质量控制到数据链路追踪的统一管理能力。

---

## 项目定位

本项目并非传统“文件上传系统”或“标注工具”，而是：

### 面向机器人与具身智能场景的数据基础设施平台

平台核心目标包括：

- 建立统一的数据资产体系；

- 支持多模态数据统一接入；

- 支持数据处理流程可追溯；

- 支持数据派生与血缘管理；

- 支持质量控制与状态管理；

- 为后续数据集构建、模型训练与机器人学习提供稳定数据底座。

平台整体采用：

```text
采集侧 + 管理侧
```

双系统架构。

---

# 一、采集侧（Acquisition Side）

采集侧定位为：

### 轻量级多模态数据采集控制系统

主要服务实验室当前真实数采需求。

核心目标：

- 统一接入多种采集设备；

- 统一开始/停止采集；

- 统一 session 与时间戳；

- 实时展示设备状态；

- 自动生成数据会话（Session）；

- 支持数据同步到管理平台。

当前重点支持三类设备：

```text
Video
HMD
IMU
```

采集流程：

```text
创建/选择采集任务
↓
连接设备
↓
统一开始采集
↓
生成 session
↓
本地缓存/保存
↓
同步到管理平台
```

采集侧重点解决：

- 设备接入；

- 实时状态监控；

- 时间同步；

- Session 管理；

- 数据标准化接入。

---

# 二、管理侧（Management Side）

管理侧定位为：

### 多模态数据生产与治理平台

主要负责数据全生命周期管理。

平台底层核心主线为：

```text
acquisition_task
    ↓
session
    ↓
data_asset
    ↓
processing_job
    ↓
derived_asset
    ↓
qc_report
    ↓
lineage
```

其中：

- `task`：业务任务；

- `session`：一次真实采集会话；

- `data_asset`：统一数据资产；

- `processing_job`：处理执行记录；

- `derived_asset`：派生数据；

- `qc_report`：质检结果；

- `lineage`：数据血缘关系。

平台所有页面均围绕：

```text
taskId
sessionId
```

两个核心上下文组织。

---

## 管理侧核心模块

### 1. 数据（核心模块）

平台核心入口。

以 `data_asset` 为主视角管理：

- 原始上传数据；

- 采集同步数据；

- 处理派生数据。

支持：

- 分页筛选；

- 元信息展示；

- 数据播放与预览；

- Session 关联；

- 处理记录查看；

- 数据链路追踪。

---

### 2. 上传

统一数据接入入口。

支持：

- 普通文件上传；

- 外部路径登记；

- MinIO / OSS / COS 等多存储源；

- 多模态数据接入。

支持上传：

```text
MCAP
BAG
BVH
MP4
CSV
SMPL
HDF5
```

等多种格式。

---

### 3. 采集

管理业务采集任务：

- task 创建；

- Session 关联；

- 被试与动作信息；

- 设备组合；

- 数据状态追踪。

---

### 4. 处理

管理数据处理流程（Pipeline）：

例如：

```text
seq → mp4
csv parse
SMPL generate
time alignment
motion reconstruction
```

平台区分：

- `processing_template`：处理规则模板；

- `processing_job`：一次实际执行。

实现处理过程可追溯。

---

### 5. 标注

当前阶段仅实现轻量标注任务管理：

- annotation_task

- annotation_status

- annotation_tag

复杂在线标注器后续扩展。

---

### 6. 质检

支持：

- 质检规则管理；

- 质检日志；

- 自动 QC 状态管理。

包括：

- 文件完整性；

- 时间戳连续性；

- 模态缺失；

- 数据异常检查。

---

### 7. 导出

导出页定位为：

### 成品数据资产视图（Final Asset）

展示：

- 已完成处理；

- 已完成标注；

- 已完成 QC；

的可交付数据资产。

而非简单下载中心。

---

# Session 核心概念

平台引入：

```text
session_id
```

作为多模态数据统一上下文。

一个 Session 下可统一关联：

```text
video
imu
csv
smpl
alignment
qc
annotation
```

Session 是平台实现：

- 多模态关联；

- 时间同步；

- 数据链路；

- 处理追踪；

的核心基础。

# 当前 MVP 场景

```text
8RGB 阵列
+
动捕服 CSV
+
SMPL
+
时间对齐
```

完整流程：

```text
创建采集任务
↓
生成 session
↓
登记 seq 外部资产
↓
上传 mp4
↓
上传 IMU CSV
↓
生成 SMPL
↓
时间对齐
↓
生成派生资产
↓
QC
↓
数据链路追踪
```

该场景已经覆盖：

- 多模态数据；

- Session 管理；

- 数据资产；

- 处理流程；

- 派生结果；

- QC；

- 数据血缘；

等平台核心能力。

# 项目目标

短期目标：

- 完成实验室真实数采链路落地；

- 建立统一数据资产体系；

- 打通采集 → 处理 → QC → 导出的主流程。

长期目标：

- 逐步扩展 EEG、足压、机器人状态等模态；

- 对接机器人学习与数据集构建；

- 面向机器人公司与代理商场景扩展；

- 建立具身智能数据生产基础设施。
