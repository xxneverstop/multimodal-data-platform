**艾欧数据开放平台**的「数据」模块界面，是机器人项目数据集的**集中管理工作台**，核心功能是对所有上传/采集的数据文件进行全生命周期管理与筛选。
---

### 一、界面核心功能拆解

#### 1. 顶部筛选区：精准定位数据

- **项目维度**：支持按项目空间筛选数据，包括「全部项目」「遥操作」「人类数据」「个人空间」，可快速区分共享/私有数据。
- **多条件筛选**：提供数据名称搜索、机器人类型筛选、标签筛选，以及「高级搜索」功能，支持组合条件精准定位目标数据。
- **状态快速过滤**：右上角的快捷筛选按钮，可按数据的分配状态（已分配/未分配）、标注状态（已标注/未标注）快速过滤列表。
  
  #### 2. 数据列表区：核心信息一览
  
  列表展示了所有数据文件的关键属性，方便批量管理：
  
  | 列名              | 说明                                                                                              |
  |:--------------- |:----------------------------------------------------------------------------------------------- |
  | **文件名**         | 数据文件名称，支持通过文件名识别数据场景（如`human_fight001.bvh`、`KitchenCutlery_DualAirbot`），部分文件带有「仓库」「点云」「低质量」等标签。 |
  | **文件/时长**       | 数据文件大小与时长，覆盖从22秒到31分钟不等的机器人/人体动作数据。                                                             |
  | **项目/上传者/上传时间** | 数据所属项目空间、上传人、上传时间，方便溯源与团队协作。                                                                    |
  | **任务/标注/质检**    | 数据的分配状态、标注条数、质检状态，可直观了解数据的处理进度。                                                                 |
  | **操作**          | 每个数据文件支持「播放」操作，可直接在平台内预览数据内容。                                                                   |
  
  #### 3. 底部工具栏：批量操作与管理
- 分页与存储信息：当前共36条数据，每页显示50条，平台总存储空间29 GB。
- 批量操作按钮：支持对选中数据进行**重命名、统计、标注、标签管理、更新、删除、导入**等批量操作，大幅提升数据处理效率。

---

### 二、模块的核心价值

1. **数据资产统一管理**：将BVH、MCAP、ROS BAG、点云等多格式数据集中管理，避免数据分散丢失，形成完整的项目数据资产库。
2. **全流程进度追踪**：从数据上传、分配任务、标注到质检，全流程状态可视化，清晰掌握每一条数据的处理进度。
3. **高效协作与筛选**：通过项目空间、标签、状态等多维度筛选，支持团队成员快速定位自己负责的数据，适配多角色协作场景。
4. **数据质量管控**：通过「低质量」标签、质检状态等字段，可快速识别并处理无效数据，保障后续模型训练的数据源质量。

### 四、整体功能与场景

该界面是机器人数据处理的核心工具，主要用于：

1. **数据回放与校验**：同时查看多传感器数据，验证采集数据的完整性与质量；
2. **数据标注**：为场景物体、机器人动作、末端位姿添加标签，构建机器人学习的训练数据集；
3. **动作分析**：结合RGB、深度、语义分割、3D位姿数据，分析机器人抓取、放置等操作的执行过程；
4. **模型训练前置**：处理后的标注数据可直接用于机器人行为克隆、强化学习等模型的训练。
   简单来说，这是一个为机器人操作任务（如抓取、分拣）打造的多模态数据标注与分析平台，通过多视图联动，实现从原始传感器数据到标注数据集的全流程处理。

这是**艾欧数据开放平台**的「上传」模块界面，核心功能是为机器人学习项目提供多格式、多存储源的数据上传与导入能力，以下是详细拆解：

---

### 一、顶部项目与存储配置区

1. **项目选择**：可切换上传目标项目空间，当前选中的是`USER_N2XNRX（个人空间）`，也支持「遥操作」「人类数据」等共享项目。

2. **存储源选择**：支持多种云存储与自建存储，当前选中**腾讯云 COS（上海地域）**，其他可选包括：
   
   - 阿里云 OSS（北京地域）
   - Amazon S3（美国西部）
   - Azure Blob Storage（美国东部）
   - Cloudflare R2（新加坡）
   - 自建云存储 MINIO

---

### 二、核心上传格式选择区

平台支持机器人学习场景的主流数据格式，当前高亮选中**BVH 文件**（人体动作捕捉数据格式），各格式的用途说明如下：

表格

| 格式类型            | 用途说明                        |
| --------------- | --------------------------- |
| **MCAP 文件**     | 机器人多模态数据推荐格式，支持高效存储传感器数据    |
| **BAG 文件**      | ROS 机器人数据的标准格式，兼容 ROS 生态工具链 |
| **HDF5 文件**     | 大容量科学数据存储格式，适合结构化数据与大型数据集   |
| **LeRobot 文件夹** | 主流机器人学习数据集格式，适配 LeRobot 生态  |
| **BVH 文件**      | 人体动作捕捉数据格式，支持可视化标注人体动作      |
| **视频文件**        | 平台可实时将视频转换为 MCAP 格式         |
| **音频文件**        | 平台可实时将音频转换为 MCAP 格式         |
| **图片文件**        | 支持静态图片的物体标注，适配视觉任务          |

此外，还提供批量上传与云端导入功能：

- **批量自动上传**：自动上传指定本地文件到平台
- **从云端导入**：从对象存储扫描并导入数据

### 四、模块整体价值

该模块是机器人数据处理流程的**数据入口**，核心作用包括：

1. **多格式兼容**：覆盖机器人、人体动作、传感器等全场景数据格式，无需提前转换；
2. **多存储支持**：适配主流云存储与自建存储，满足不同团队的部署需求；
3. **自动化处理**：支持视频 / 音频自动转 MCAP，降低数据预处理成本；
4. **项目化管理**：上传数据直接关联指定项目空间，无缝衔接后续采集、标注、质检流程。
